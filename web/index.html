HookDump

A keyboard / mouse logger for Windows 3.x – 95/98. Object Pascal. Nizhny Novgorod, 1996–1997. One of the best-known Russian "spy programs" of its day — preserved as a historical software artifact.

HookDump installs a global Windows keyboard hook (SetWindowsHookEx(WH_KEYBOARD, …)) from a DLL and records to a file everything typed on the keyboard and clicked with the mouse while it runs in the background, noting for each event which program, window and field it occurred in. It grew out of an earlier program by the same author — HookRus, a Russian keyboard-layout patcher for Windows. It has its own articles in the Russian and Ukrainian Wikipedias.

The source code is released under the MIT License so anyone can study it. The program does not run on modern systems — it was a 16-bit Windows program (3.x, and the Win16 subsystem of 95/98). Its system-wide hook relied on the 16-bit shared-address-space multitasking model; once Windows moved to 32-bit isolated per-process address spaces it could no longer see other programs, and it does not run on today's 64-bit Windows. Kept for historical and educational study only.

Author's intent. HookDump was never meant as a destructive or malicious tool. It was a monitor — built to log how and when the computer it was installed on was used. It was originally written for a university department, on commission from the head of the department, to track the time and purpose of use of the department's office computers.
In the 1990s this was described, in the style of the time, as a "program-spy." By modern definitions it is a keylogger. It is published here as a record of early work and for study. Do not run the historical binaries on a live system, and never capture anyone's input without their consent and lawful authority.

Why it mattered

  • Invisible to the user and to the Windows Task Manager while running.
  • Context logging — captured the active program, window title and field for each event, including hidden Dial-Up Networking passwords.
  • Cross-process shared state via named file mapping (CreateFileMapping/MapViewOfFile), so the DLL injected into every process could share the hook and window handles with the controller.
  • Configurable down to which keys are logged; optional mouse logging, XOR obfuscation of the log, write buffering, and an AutoStartUp option.
  • Dual use — later cited in a textbook as a way to collect interface-usage statistics, i.e. UX analytics before the term was common.

How it worked

  • Embeds itself into the system and becomes invisible to the user, writing every action — keystrokes, mouse clicks, and the program/window/field context — to a file.
  • The log file is created automatically in a chosen directory (or the Temp directory) with a unique or specified name; recording begins right after the OS starts.
  • Writes happen immediately or are buffered; the file's normal state is closed.
  • To uninstall, relaunch in visible mode with the /V switch, then exit.

Configuration — HOOKDUMP.INI

[StartUp]
Show=00          ; start hidden
Dir=C:\          ; log directory; if absent, the Temp dir is used
file=_Hook       ; base file name; else a unique name with extension .hk
XorBase=85       ; XOR key when XorOutput is on
BufSize=255      ; write buffer size

[Options]
WriteBasicKey=01   WriteExeName=01    AnsiToOem=00   XorOutput=00
WritePopKey=00     WriteWindowName=01 RegardMouse=01 UseBuffer=01
WritePushKey=00
